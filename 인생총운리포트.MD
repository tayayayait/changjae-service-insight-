# 인생 총운(정통) 7리포트 구성안 v2.1

## 설계 원칙
- 공통 앵커: 모든 리포트는 `현재 시점`, `현재 나이(만 나이)`, `현재 흐름`에서 시작한다.
- 시간 해상도 분리: 같은 시점을 공유하되 리포트 목적에 맞게 구간 단위를 다르게 쓴다.
- 완성형 분석 블록: 모든 구간은 아래 5요소를 필수로 포함한다.
  - 핵심 흐름
  - 해석 근거
  - 주요 기회
  - 리스크
  - 행동 전략
- 차별화 원칙: 공통 맥락 문장 1개를 제외하고, 서비스 간 핵심 문장을 반복하지 않는다.

---

## 1) 인생 총운 로드맵
### 핵심 질문
- 내 인생 흐름이 어떤 단계로 전개되는가?

### 시간축
- 직전 / 현재 / 다음 / 다다음 (10년 x 4구간)

### 반드시 보여줄 내용
- 각 시기별 인생 단계 변화
- 직전 시기에서 현재 시기로 이어지는 전개 방향
- 시기별 성숙/확장/정리 축
- 단편 운세가 아닌 인생 서사

### 강조 포인트
- 핵심 흐름
- 전개 방향
- 인생 단계 변화

### 블록 출력 템플릿
- 구간: {windowLabel} ({timeRange})
- 핵심 흐름: {coreFlow}
- 해석 근거: {evidence}
- 주요 기회: {opportunities[]}
- 리스크: {risks[]}
- 행동 전략: {actionStrategy[]}

## 차트 근거형 계약 (재물/에너지)
- 목적: 차트 수치와 해석을 모두 만세력 원데이터와 연결해 사용자 혼동을 제거한다.
- 수치 생성 원칙:
  - `saju-wealth-flow.assetTrendSeries`와 `saju-energy-balance.energyRhythmSeries`는 모델 생성값을 사용하지 않는다.
  - 시스템 deterministic 엔진이 `원국 오행 + 용신 + 세운(연간 간지)`으로 계산한다.
- 근거 노출 원칙:
  - 각 포인트는 `assetTrendEvidence[]` / `energyRhythmEvidence[]`를 반드시 가진다.
  - evidence는 `시점, 지수, 변화폭, 방향, 원인 요약, 행동 해석, reasonDetails, rawBasis`를 포함한다.
  - `rawBasis.source`는 항상 `"manseryeok"`이다.
- UI 원칙:
  - 차트 상단: `만세력 원국/용신/세운 기반 체크포인트 지수` 한 줄 설명 고정.
  - 차트 하단: `근거 표` 고정 렌더 (`시점/지수/변화폭/상승·하락 원인/해석`).
  - 각 행은 `근거 보기` 확장에서 `오행 분포/용신/세운(또는 세운·월운·일진)/요인 점수`를 표시한다.
