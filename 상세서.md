# 사주·운세 웹 플랫폼 UI/UX 구현 상세서

## 개요

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
| --- | --- | --- | --- |
| 문서 목적 | `MVP UI/UX 기준서 1종` | 전체 제품 | 디자인, 프론트엔드, QA가 동일 기준으로 사용한다. 개별 해석으로 수치를 바꾸지 않는다. |
| 디자인 원칙 01 | `모바일 퍼스트` | 전체 화면 | 설계 시작점은 `360px`이다. 데스크톱은 같은 정보 구조를 확장한다. |
| 디자인 원칙 02 | `한자 없는 해석` | 결과 카드, 도움말, 입력 가이드 | 전문 용어는 최초 1회만 쉬운 한국어 설명과 함께 노출한다. 한자 단독 표기는 금지한다. |
| 디자인 원칙 03 | `감성적 카드 중심` | 홈, 리포트, 운세, 궁합 | 핵심 정보는 카드 단위로 묶는다. 한 화면에 1차 강조 카드 수는 `3개`를 넘기지 않는다. |
| 디자인 원칙 04 | `정보 우선 시각화` | 오행 차트, 대운 타임라인, 운세 그래프 | 차트 앞에 요약 수치와 한 줄 해석을 먼저 노출한다. 차트만 단독 배치하지 않는다. |
| 범위 포함 | `홈 / 사주 입력 / 결과 리포트 / 운세 / 궁합 / 마이페이지` | MVP 화면 | 빈 상태, 로딩 상태, 오류 상태, 저장·공유 CTA까지 포함한다. |
| 범위 제외 | `명리 계산 엔진 / 로그인 API / 결제 SDK` | 백엔드, 외부 연동 | UI가 기대하는 입력값, 출력 상태, 실패 메시지 규칙만 정의한다. |

## 디자인 토큰

### 토큰 네이밍 규칙

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
| --- | --- | --- | --- |
| 색상 토큰 | `color.*` | Figma 변수, CSS 변수 | 의미 기반으로 관리한다. 예: `color.bg.base`, `color.text.primary`. |
| 타이포 토큰 | `font.*` | 텍스트 스타일 | 크기, 줄간격, 굵기를 한 세트로 관리한다. 예: `font.body`. |
| 여백 토큰 | `space.*` | padding, gap, margin | 임의 숫자 사용 금지. 표에 없는 여백은 가장 가까운 토큰으로 반올림한다. |
| 라운드 토큰 | `radius.*` | 카드, 입력창, 모달 | 원형 버튼 외에는 `12px`, `16px`, `24px`만 사용한다. |
| 그림자 토큰 | `shadow.*` | 카드, 모달, 드로어, 토스트 | 한 컴포넌트에 그림자 레이어는 1개만 사용한다. |
| 모션 토큰 | `motion.*` | hover, page transition, modal | `transform`, `opacity`, `box-shadow`만 애니메이션한다. `width`, `height` 애니메이션은 금지한다. |
| 레이어 토큰 | `z.*` | sticky, overlay, toast | 표의 값만 사용한다. 임의 `z-index` 추가 금지. |
| 브레이크포인트 토큰 | `breakpoint.*` | 반응형 분기 | `360`, `768`, `1024`, `1440` 기준만 사용한다. |

| 고정 요소 겹침 | `GNB 64px / 탭 72px / CTA 80px` | sticky, fixed 영역 | 본문, 토스트, 모달이 고정 요소에 가려지지 않아야 한다. |
| 상태별 시각 일관성 | `default / hover / active / focus / disabled / loading / error` | 8개 컴포넌트 전종 | 모든 상태가 누락 없이 구현되고 동일 토큰 체계를 따라야 한다. |
| 긴 텍스트 줄바꿈 | `2줄 clamp 또는 자동 줄바꿈` | 카드 제목, 도움말, 토스트 | 영문, 숫자, 긴 한국어 문장에서 overflow가 없어야 한다. |
| 빈 상태 검수 | `icon 32px / body 14px / CTA 40px` | 카드, 표, 목록 | 데이터가 없을 때 빈 화면이 아니라 안내형 empty state를 노출한다. |
| 오류 상태 검수 | `error soft + retry` | 카드, 필드, 모달, 토스트 | 오류 사유와 재시도 액션이 동시에 보여야 한다. |
| 로딩 상태 검수 | `skeleton / spinner` | 목록, 차트, 버튼 | 로딩 중 레이아웃 점프가 없어야 한다. |
| 차트 범례 검수 | `label 13px / gap 8px` | 오행, 대운, 운세 그래프 | 범례가 색상+텍스트+수치를 함께 표시하는지 확인한다. |
| 저장·공유 CTA 노출 | `56px ~ 80px` | 결과 리포트, 궁합 | 결과 하단까지 스크롤하지 않아도 저장 또는 공유 CTA에 접근 가능해야 한다. |
| 모달/드로어 닫기 동작 | `Esc / 닫기 버튼 / 바깥 탭` | 오버레이 전종 | 허용된 닫기 수단이 일관돼야 한다. 차단 상태에서는 예외를 명시한다. |
| 입력 유효성 검수 | `error 14px / ring 2px` | 사주 입력, 궁합 입력 | 빈 값, 잘못된 시간, 미지원 날짜 등 경계값을 모두 확인한다. |
| safe-area 검수 | `env(safe-area-inset-*)` | iOS 모바일 | 하단 탭, sticky CTA, 토스트가 홈 인디케이터와 겹치지 않아야 한다. |
| 접근성 검수 | `Tab / SR / Contrast` | 전체 UI | 키보드 탐색, 스크린리더 읽기, 명암 대비를 릴리즈 전 점검한다. |

## 가정(Assumptions)

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
| --- | --- | --- | --- |
| 범위 기준 | `MVP` | 전체 문서 | 프리미엄 심층 리포트, AI 상담, 알림, 게이미피케이션은 확장 여지로만 본다. |
| 플랫폼 | `모바일 웹 우선 반응형 웹` | 전체 제품 | 네이티브 앱 전용 제스처와 OS 전용 UI 패턴은 본 문서 범위에서 제외한다. |
| 아이콘 시스템 | `선형 SVG / 20px / 24px` | 전체 UI | 이모지 아이콘은 사용하지 않는다. |
| 토큰 사용 방식 | `Figma Variables + CSS Custom Properties` | 디자인, 프론트엔드 | 토큰 이름은 문서 표기를 그대로 사용한다. |
| 차트 구현 방식 | `라이브러리 무관` | 결과 리포트, 운세 | 색상, 여백, 범례, 접근성 기준만 고정하고 라이브러리 선택은 구현 단계에서 결정한다. |
| 날짜·시간 선택기 | `Input + Drawer 조합` | 사주 입력, 궁합 입력 | 모바일 휠 선택 경험을 우선한다. 브라우저 기본 picker는 보조 옵션으로만 본다. |
| 기본 정렬 규칙 | `최신순 / 중요도 우선` | 저장 목록, 알림 목록 | 사용자가 별도 정렬을 바꾸지 않으면 최신 결과를 먼저 보여준다. |
