React/Vite 작업 전 사전 점검 보고서

작성일: 2026-03-24 (Asia/Seoul)

기준: Vercel React Best Practices (우선순위 8개 카테고리)

1. 현재 상태(확인된 팩트)

앱 구조
Vite + React 18 + TypeScript

테스트
Vitest (test 스크립트 존재)

린트
ESLint (lint 스크립트 존재)

Next.js 사용
코드/의존성 기준 직접 사용 흔적 없음

주의: Next.js 전용 규칙(server-*, 일부 rendering-*)은 현재 구조에서 직접 적용 범위가 제한됩니다.

2. 작업 전 필수 체크리스트

우선순위 점검 항목 실행 방법 완료 기준
P0 의존성 동기화 pnpm install 설치 에러 0건
P0 기준선 품질 확인 pnpm lint, npm test 린트/테스트 실패 0건
P0 비동기 워터폴 탐지 (async-*) 순차 await 구간 식별 후 병렬화 후보 목록 작성 병렬화 대상 목록 확정
P1 번들 최적화 후보 식별 (bundle-*) 무거운 UI/차트/애니메이션 모듈의 지연 로딩 후보 정의 동적 import 적용 대상 확정
P1 클라이언트 데이터 패턴 점검 (client-*) 중복 fetch, 전역 이벤트 리스너, 로컬스토리지 스키마 점검 중복 요청/리스너 제거 계획 작성
P2 재렌더/렌더링 비용 점검 (rerender-*, rendering-*) 불필요한 상태 구독, 인라인 컴포넌트, 과도한 memo 사용 확인 수정 우선순위 리스트 확정

3. 적용 제외/제한 항목

  • Next.js 전용 API 또는 App Router 관련 항목: 현재 구조 기준 직접 적용 불가.
  • 서버 액션 인증(server-auth-actions) 등 서버 프레임워크 종속 규칙: 현재 리포지토리의 프런트엔드 범위에서는 제한적.
  • 추가 서버 구조 확인 전, 서버 전용 성능 규칙의 효과를 단정할 수 없음.

4. 착수 게이트(승인 조건)

  • 기준선 테스트(npm test) 통과
  • 린트(pnpm lint) 통과
  • 워터폴/번들 최적화 후보 목록 작성 완료
  • 변경 대상 문서(docs/) 업데이트 계획 확정
본 보고서는 작업 착수 전 체크리스트 용도이며, 성능 수치 자체는 별도 측정 결과가 필요합니다.
